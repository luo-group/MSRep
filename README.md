## Install Dependencies

```bash
conda create -n msrep python=3.8
conda activate msrep
pip install -r requirements.txt
```

## Run MSRep to Predict Function Annotations

### Construct Test and Lookup Databases

Organize the test proteins in a CSV file with columns `Entry` and `Sequence`. See `data_cleaned/EC/swissprot_test_ec.csv` as an example.

First, construct test and lookup databases with ESM embeddings:

```bash
python scripts/extract_esm.py -i /path/to/test_csv -o /path/to/test_pt --ont ec
python scripts/extract_esm.py -i /path/to/lookup_csv -o /path/to/lookup_csv --ont ec
```

Arguments:

- `-i`: input test CSV file
- `-o`: output test `pt` file
- `--ont`: function of interest. Available choices are `ec`, `gene3D`, `pfam`, `BP`, `MF`, and `CC`.

You can use the MSRep training data as lookup data, available in the `data` directory for each function annotation task, such as `data_cleaned/EC/swissprot_train_ec.csv`. You can also use your own lookup data organized in a CSV file with columns `Entry`, `Sequence`, and `Label`.

### Inference

Run MSRep with the following command:

```bash
python scripts/predict.py configs/infer.yml
```

Parameters are set in the configuration file:

- `model_dir`: path to the pretrained model directory, available in the `checkpoints` directory for each task. We provide five pretrained models with different random seed initialization for each task.
- `lookup_data`: path to the lookup `pt` file
- `test_data`: path to the test `pt` file
- `label_file`: list of all labels for the function, available as a JSON file in the `data` directory for each function
- `ont`: function of interest. Available choices are `ec`, `gene3D`, `pfam`, `BP`, `MF`, and `CC`.

### Ensemble

Run majority-vote ensembling for a list of prediction results with the following command:

```bash
python scripts/ensemble_majority_voting.py configs/ensemble.yml
```

Parameters are set in the configuration file:

- `prediction_files`: list of paths to the prediction results generated by MSRep

### Post-Inference Processing for Gene Ontology

Following the convention of the CAFA challenge, we process Gene Ontology prediction results after inference.

First, download the GO hierarchy file:

```bash
wget http://purl.obolibrary.org/obo/go/go-basic.obo
```

Then conduct backpropagation on the original prediction result made by MSRep:

```bash
python scripts/flatten_GO.py -i /path/to/prediction -o /path/to/flat_prediction
python scripts/backprop.py --go_obo /path/to/go-basics.obo -i /path/to/flat_prediction
```

To ensemble the backpropagated predictions, run:

```bash
python scripts/ensemble_GO.py configs/ensemble_GO.yml
```

Parameters are set in the configuration file:

- `prediction_files`: list of paths to the backpropagated prediction results generated by MSRep

## Train MSRep

You can train new MSRep models with your own data. First construct training `.pt` files following the commands in [Construct Test and Lookup Databases](#construct-test-and-lookup-databases). Then run:

```bash
python scripts/train_MSRep.py /path/to/config
```

Example configuration files for training on different function annotation tasks are available in the `configs` directory in the format `train_*.yml`. Replace the dataset paths with the generated `pt` data files.
